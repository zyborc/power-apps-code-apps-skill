# setup.md — Project Init & Structure

---

## ⚠️ Anti-Loop Rules — Read First

These rules exist to prevent Claude from draining quota in a broken init cycle:

1. **Never manually recreate the Power Apps project structure.** If the scaffold is missing, run `pac code init` — do not copy or generate files by hand.
2. **Never move or rename files as a "fix".** If a build fails, diagnose the error message and address the root cause.
3. **If `pac code push` fails because the app isn't registered:** the app must first exist in Power Apps. Create it via the Power Apps portal or `pac code init`, then push.
4. **Stop and ask the user** if more than one CLI command fails in sequence. Do not attempt workarounds autonomously.

---

## 0. Choose Your Init Path

```
Starting a new project?
│
├─ Have the org template repo? → Section 1 (Template)
│
└─ Starting completely from blank? → Section 2 (From Scratch)
```

---

## 1. Start from Template (preferred)

```bash
git clone https://your-org/your-powerappscode-template.git .
npm install
```

After cloning, skip to **Section 3 (Authenticate)**.

---

## 2. Start from Scratch (`pac code init`)

Use this path when there is no template or the user explicitly wants a blank project.

```bash
# 1. Authenticate first (required before init)
pac auth create --environment https://<org>.crm.dynamics.com

# 2. Create a new empty folder and cd into it
mkdir <app-name> && cd <app-name>

# 3. Scaffold the Power Apps Code App project
pac code init --name <AppName>

# 4. Install dependencies
npm install
```

> `pac code init` creates the correct `power.config.json`, `vite.config.ts`, and base structure.
> **Do not skip this step and try to build manually — it will fail.**

After init, verify the canonical `package.json` versions match **Section 4** below, and update if needed.

---

## 3. Authenticate

```bash
pac auth create --environment https://<org>.crm.dynamics.com
pac auth list                        # verify active profile
pac auth select --index <n>          # switch environments
pac env select --environment <id>    # set target environment
```

## 5. Folder Structure

```
src/
├── App.tsx              ← SDK init, routing, user/role context
├── main.tsx             ← React entry point only
├── components/
│   ├── layout/          ← Sidebar, PageHeader, Shell
│   ├── ui/              ← reusable primitives (Button, Badge, Card...)
│   └── views/           ← one file per page/view
├── hooks/               ← all data access lives here
├── types/
│   └── index.ts         ← shared TypeScript interfaces
└── generated/           ← AUTO-GENERATED by pac CLI — never edit
    ├── models/
    └── services/
```

**Rules:**
- `generated/` is owned by PAC CLI — never touch manually
- All data fetching in `hooks/` — never inside components
- One view per file — no mega-components

## 6. Enable Code Apps (Admin — once per environment)

```
Power Platform Admin Center
→ Environments → select env
→ Settings → Product → Features
→ Toggle "Power Apps code apps" ON → Save
```

End-users require **Power Apps Premium license**.

---

## 7. Deploy

```bash
npm run build    # compile
pac code push    # publish to Power Platform
```
